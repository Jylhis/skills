# Research

Investigate a topic, technology, library, or approach. Produce a summary the user can make decisions from.

## Steps

1. Clarify the research question. If the user said something vague like "look into X", ask what specifically they need to decide or understand.
2. Search docs, repos, issues, and community sources. Use web search and documentation tools when available.
3. For each option or finding, note: what it is, who uses it, trade-offs, and maturity level.
4. Form a recommendation. State what you would pick and why.

## Output

Structure the summary as:
- **Question**: what we set out to answer
- **Findings**: what exists, with trade-offs for each option
- **Recommendation**: what to use and why
- **Sources**: links to docs, repos, or discussions

Keep it short. The user wants to make a decision, not read a textbook.
