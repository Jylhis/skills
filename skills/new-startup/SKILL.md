# New Startup

Go from startup idea to a validated concept and, if it holds up, an initial project.

## Steps

1. Ask the user to describe the idea in one sentence. Who has the problem? How painful is it?
2. Challenge the idea with these questions:
   - Who is the most desperate customer? Name them specifically (role, company size, situation).
   - What do they do today without your product? How much does that cost them (time, money, pain)?
   - Why hasn't someone built this already? If they have, why is yours better?
3. Research the competitive landscape. Find existing solutions, adjacent products, and failed attempts. Note what worked and what didn't.
4. Define the narrowest wedge: the smallest possible thing that proves real people will pay or show up.
5. Outline a v1 scope that tests the core hypothesis. If you can't ship it in a few weeks, the scope is too big.
6. If the user wants to proceed, scaffold the project (same flow as `/new-project`).

If the idea doesn't hold up at any step, say so directly. Saving weeks of wasted effort is more valuable than being encouraging.

## Output

- **Idea assessment**: the honest take on whether this has legs
- **Competitive landscape**: what exists, what's missing
- **Wedge**: the smallest thing to build first
- **V1 scope**: what to ship, what to skip
